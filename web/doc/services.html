Generic

There are some generic parameters:

  • &session=<token>, (Mandatory)To identify the user
  • &html=on, (Optional) to set the content type to html to be able to easily load in the browser (for testing)
  • &pp=on, (Optional) to add pretty printing to easier be able to read the json output(for testing)

Description of services

Login:

Note: Login is not implemented. The service will return a static token "DummySessionToken"

GET Request:

/Login?user=<name>&password=<password>

< >

Response json:

{ "Session" : "<key>" }

Error management:

No error handling implemented

Project:

Request a list of projects

GET Request:

/Project?session=<token>&key=<key>

Response json:

{ Project : [ { "id" : "<key>", "name" : "<project name>", "description" : "<description text>"} ] }

Other examples:

/Project -- returns all projects

POST Request:

The post request will either create a new entry or update an existing entry. For the update operation, all parameters has to be provided.

/Project?session=<token>&name=<name>&description=<description>

Response json:

{ "Project" : <key> }

The response contains the id to the created/updated project entry

Error management:

If the session token does not match an active token, the service will return 403 - forbidden.

Other examples:

/Project?session=<token>&key=<key>&name=<name>&description=<description> -- will update an existing project entry

Document:

Request a list of documents for a specific project

NOTE: This request will return the project as a name, not an id. (change?)

GET Request:

/Document?session=<token>&project=<key>

Response json:

{ Document : [ { "id" : "<key1>", "name" : "<document name1>", "description" : "<description text>", "project" : "<project name>"}, { "id" : "<key2>", "name" : "<document name2>", "description" : "<description text>", "project" : "<project name>"} ] }

Error management:

If the session token does not match an active token, the service will return 403 - forbidden.

Other examples:

/Document?session=<token> -- returns all documents

/Document?session=<token>&document=<key> -- returns a specific document

Fragment:

Get all fragments related to a project

GET Request:

/Fragment?session={token}&project={key}

Response json:

{ Fragment : [ { "id" : "<key1>", "Text" : "<fragment text1>", "Contract" : "<Contract name>", "project" : "<project name>"}, { "id" : "<key2>", "Text" : "<fragment text2>", "Contract" : "<Contract name>", "project" : "<project name>"} ] }

Error management:

If the session token does not match an active token, the service will return 403 - forbidden.

Other examples:

/Fragment?session=<token>&key=<key> -- returns a specific fragment

Keywords:

Get all keywords related to a project. This should be used for the auto complete field in the search

GET Request:

/Keywords?session=<token>&project=<key>

Response json:

{ "Keywords" : [ { "keyword" : "<word>"} { "keyword" : "<word>"} ]}

Error management:

If the session token does not match an active token, the service will return 403 - forbidden.
